Zero-Copy Audio

Audio samples flow through the pipeline using Arc-wrapped buffers. No redundant memory copies from microphone to inference.

Event-Driven

Decoupled components communicate via typed events. Turn detection, VAD, and transcription operate independently.

Clean Architecture

Domain logic separated from infrastructure. Engine loaders implement traits for dependency inversion.

Design Decisions

Why we built it this way.

Runtime

Rust + Tauri over Electron

Native performance with a ~15MB binary instead of bundling an entire browser. Tauri v2 provides secure IPC between the Rust backend and React frontend. Memory-safe concurrency for audio processing without garbage collection pauses.

Inference

ONNX Runtime over PyTorch

Single inference runtime supporting multiple model architectures. Hardware acceleration via CoreML on Apple Silicon. No Python dependency ‚Äî pure Rust bindings for minimal overhead.

Audio

cpal for cross-platform capture

Low-level audio capture with direct access to system audio devices. Ring buffer architecture prevents memory allocation during recording. Consistent 16kHz resampling regardless of hardware sample rate.

STT Engine

sherpa-onnx for production inference

Battle-tested C++ inference engine with Rust bindings. Supports streaming transducers, CTC, and Whisper architectures. Built-in voice activity detection and endpoint detection.

State

Plugin isolation with shared state

Each Tauri plugin manages its own domain (recorder, STT, tools). Shared audio bus enables zero-copy data flow between plugins. Event system prevents tight coupling while maintaining type safety.
